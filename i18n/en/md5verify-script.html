While batch looks like doesn’t have any good encryption and can be easily reversed, but you can try it if you need to.

Calculating the MD5 value requires the certutil tool, which seems to be built-in since Win7, if you can’t get it to work, please leave a comment

Parse

In this case, the variable password, which represents the password, is first set to the MD5 value of 12345 in the .iL file
It then enters the loop :PassCheck, where the user is asked to enter a password, which is saved to the variable %typestr% and to the temporary file %temp%\pass.iL
Then use certutil to calculate the MD5 value of the temporary file %temp%\pass.iL, filter and save the result to the variable %typemd5% and delete the temporary file
It is then compared to the hash of the preset password, and if the two match, jump to :next and continue with the rest of the commands
If it doesn’t match, call :PassCheckError to display the “Incorrect password” message and goes back to the beginning
